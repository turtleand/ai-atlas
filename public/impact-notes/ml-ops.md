# ML Ops
**Status:** Frontier

## Why this area is shifting
Managed AI services (OpenAI API, Anthropic API, AWS Bedrock) abstract away much of the traditional MLOps stack. You no longer need to manage training infrastructure, model serving, or GPU clusters for many use cases. But custom model training, fine-tuning pipelines, and production reliability engineering still require skilled practitioners.

## Timeline
- **2024-01** - API-first AI services reduce need for in-house ML infrastructure
- **2024-06** - Managed fine-tuning (OpenAI, Anthropic) simplifies custom model workflows
- **2025-01** - "MLOps engineer" roles shift toward "AI platform engineer" focused on orchestration
- **2025-07** - Open-source model deployment becomes plug-and-play with tools like vLLM and Ollama
- **2025-12** - Traditional MLOps (training pipelines, feature stores) commoditizes; focus shifts to evaluation and reliability
