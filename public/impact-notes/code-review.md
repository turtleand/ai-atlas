# Code Review
**Status:** Frontier

## Why this area is shifting
AI tools can catch common code review issues: style violations, potential bugs, security vulnerabilities, and performance problems. They work faster than humans for surface-level review. But nuanced feedback about architecture, readability, and team conventions still requires human judgment.

## Timeline
- **2024-02** - CodeRabbit and other AI review bots gain traction on GitHub PRs
- **2024-07** - Amazon CodeGuru and similar tools handle security-focused code review
- **2025-01** - AI reviewers catch 60-80% of issues that human reviewers flag
- **2025-08** - Teams adopt AI as "first pass" reviewer, with humans doing architectural review
- **2026-01** - Standard code review for style and bugs is largely automated; human review focuses on design
